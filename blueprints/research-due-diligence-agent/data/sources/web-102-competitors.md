# Industry brief — competitive landscape for vector databases

Source: stubbed offline "web search" (mock competitive scan)
Doc type: web
Date: 2025

The vector database competitive landscape splits into three groups. First, dedicated
managed vendors that sell a purpose-built vector store as a service; Acme competes
directly here. Second, general-purpose databases that have added a vector index as a
feature, letting teams avoid running a separate system. Third, open-source libraries
that teams self-host, trading operational burden for zero license cost.

The most-cited competitive pressure on dedicated vendors is the "good enough" vector
index now shipping inside incumbent databases. For many mid-market teams a built-in
index removes the need to adopt a separate product, which compresses pricing for the
dedicated vendors and raises the bar on differentiation.

Acme's stated differentiation is hybrid search quality and a serverless cost model.
Buyers in this category increasingly evaluate vendors on retrieval quality measured
by recall and on total cost at scale, not on raw vector count.
